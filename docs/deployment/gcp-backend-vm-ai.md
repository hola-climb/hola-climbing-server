# GCP Backend + VM AI Deployment

This runbook describes the production deployment path for the Spring backend on Cloud Run and the VM-hosted PostgreSQL, Redis, and AI worker stack.

## Source Of Truth

- GitHub Actions workflow: `.github/workflows/deploy-cloud-run.yml`
- Non-sensitive runtime values: GitHub repository variables
- Sensitive runtime values: GCP Secret Manager secrets referenced by the Cloud Run deploy step
- Manual Cloud Run console env changes are temporary. The next GitHub Actions deploy overwrites them.

## Monthly Report LLM Configuration

Monthly reports should be pre-generated by the scheduler in production. App reads should normally return cached rows from `monthly_reports`.

GitHub repository variables:

| Variable | Production value |
|---|---|
| `MONTHLY_REPORT_MIN_VIDEOS` | `3` |
| `MONTHLY_REPORT_MIN_PROBLEMS` | `10` |
| `MONTHLY_REPORT_GENERATE_ON_MISS` | `false` |
| `MONTHLY_REPORT_PROMPT_VERSION` | `monthly-report-v1` |
| `MONTHLY_REPORT_SCHEDULER_ENABLED` | `true` |
| `MONTHLY_REPORT_SCHEDULER_CRON` | `0 0 0 1 * *` |
| `MONTHLY_REPORT_LLM_MODE` | `openai` |
| `MONTHLY_REPORT_LLM_BASE_URL` | `https://api.openai.com/v1` |
| `MONTHLY_REPORT_LLM_MODEL` | `gpt-4.1-mini` |
| `MONTHLY_REPORT_LLM_TIMEOUT_SECONDS` | `20` |

GCP Secret Manager secret:

| Cloud Run env | Secret Manager secret |
|---|---|
| `MONTHLY_REPORT_LLM_API_KEY` | `hola-openai-api-key:latest` |

## Set GitHub Repository Variables

The installed `gh` version may not have `gh variable set`, so use the REST API through `gh api`.

```bash
REPO=hola-climb/hola-climbing-server

set_repo_var() {
  local name="$1"
  local value="$2"
  gh api --method PATCH "repos/${REPO}/actions/variables/${name}" \
    -f name="$name" \
    -f value="$value" >/dev/null 2>&1 || \
  gh api --method POST "repos/${REPO}/actions/variables" \
    -f name="$name" \
    -f value="$value" >/dev/null
}

set_repo_var MONTHLY_REPORT_MIN_VIDEOS 3
set_repo_var MONTHLY_REPORT_MIN_PROBLEMS 10
set_repo_var MONTHLY_REPORT_GENERATE_ON_MISS false
set_repo_var MONTHLY_REPORT_PROMPT_VERSION monthly-report-v1
set_repo_var MONTHLY_REPORT_SCHEDULER_ENABLED true
set_repo_var MONTHLY_REPORT_SCHEDULER_CRON "0 0 0 1 * *"
set_repo_var MONTHLY_REPORT_LLM_MODE openai
set_repo_var MONTHLY_REPORT_LLM_BASE_URL https://api.openai.com/v1
set_repo_var MONTHLY_REPORT_LLM_MODEL gpt-4.1-mini
set_repo_var MONTHLY_REPORT_LLM_TIMEOUT_SECONDS 20
```

If `gh auth status` reports an expired token, re-authenticate first:

```bash
gh auth login -h github.com
```

## Add The OpenAI API Key Secret

Do not put the OpenAI API key in GitHub variables, workflow YAML, local docs, or committed `.env` files.

```bash
PROJECT_ID=hola-climbing-log
REGION=asia-northeast3
SERVICE=hola-backend
SECRET_ID=hola-openai-api-key

gcloud secrets describe "$SECRET_ID" --project="$PROJECT_ID" >/dev/null 2>&1 || \
  gcloud secrets create "$SECRET_ID" \
    --project="$PROJECT_ID" \
    --replication-policy=automatic

read -rsp "OpenAI API key: " OPENAI_API_KEY
printf '\n'
printf '%s' "$OPENAI_API_KEY" | \
  gcloud secrets versions add "$SECRET_ID" \
    --project="$PROJECT_ID" \
    --data-file=-
unset OPENAI_API_KEY
```

Grant the Cloud Run runtime service account access to the secret:

```bash
PROJECT_ID=hola-climbing-log
REGION=asia-northeast3
SERVICE=hola-backend
SECRET_ID=hola-openai-api-key

RUNTIME_SA="$(gcloud run services describe "$SERVICE" \
  --project="$PROJECT_ID" \
  --region="$REGION" \
  --format='value(spec.template.spec.serviceAccountName)')"

if [ -z "$RUNTIME_SA" ]; then
  PROJECT_NUMBER="$(gcloud projects describe "$PROJECT_ID" --format='value(projectNumber)')"
  RUNTIME_SA="${PROJECT_NUMBER}-compute@developer.gserviceaccount.com"
fi

gcloud secrets add-iam-policy-binding "$SECRET_ID" \
  --project="$PROJECT_ID" \
  --member="serviceAccount:${RUNTIME_SA}" \
  --role="roles/secretmanager.secretAccessor"
```

## Deploy

After the variables and secret are ready, run the Cloud Run workflow:

```bash
gh workflow run "Deploy Cloud Run" --repo hola-climb/hola-climbing-server --ref master
```

Verify the latest run:

```bash
gh run list --repo hola-climb/hola-climbing-server --workflow "Deploy Cloud Run" --limit 5
```

## Smoke Checks

1. Check Cloud Run env includes `MONTHLY_REPORT_LLM_MODE=openai` and scheduler variables.
2. Check logs do not repeatedly show `monthly report LLM narrative generation failed`.
3. Call `GET /api/stats/me/monthly-reports?month=YYYY-MM` with a user that has enough analyzed videos and logs.
4. Confirm insufficient users return `insufficientData`.
5. Confirm repeated reads return the stored monthly report instead of generating new output.
